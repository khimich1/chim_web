# Конструктор заданий преподавателя

**Проект:** `chim_web`  
**Дата:** 2026-06-20  
**Статус:** идея согласована → формализация в [`SPEC.md`](../../SPEC.md) §1.9, план — Phase 14 (`tasks/plan.md`, Tasks 66–74)

---

## Problem Statement

**Как дать преподавателю создавать собственные темы с заданиями (текст, число, картинки), чтобы все его ученики видели их в разделе «Тесты», могли проходить самостоятельно, а преподаватель — назначать как домашнее задание с ручным выбором вариантов из банка ЕГЭ/ОГЭ?**

Сейчас тесты и ДЗ работают только с готовым банком экзамена (`test_ege.db` / `test_oge.db`). Преподаватель может собрать ДЗ из лекций и типовых тестов (`HomeworkForm`), но не может создать свой контент. У ученика вкладка «По заданиям» показывает только номера из банка ЕГЭ (`TaskTypePicker`), без тем преподавателя.

---

## Recommended Direction

Вертикальные срезы **A → B → C**. Custom content в PostgreSQL + файловое хранилище картинок. Переиспользование существующего UX пошаговых тестов (`TestSession`, `StepView`, `StepProgressDots`).

### Принципы

- **Два типа задания** — преподаватель выбирает при создании:
  - `auto` — числовой/текстовый ответ, авто-проверка (как в ЕГЭ).
  - `self_check` — письменное; у ученика кнопка **«Сравнить ответ»** показывает **весь эталон** (текст + картинки + число).
- **Темы** — контейнер для заданий; все ученики преподавателя видят опубликованные темы.
- **Единый TestSession** — расширить `source` (`exam` | `custom`), чтобы не дублировать Stepik-flow.
- **Редактирование без версий** — задание можно менять всегда; ученики видят актуальную версию (активная сессия может показывать старый текст до перезагрузки — приемлемо для MVP).

### UX-точки интеграции

| Место | Изменение |
|-------|-----------|
| Кабинет преподавателя | Новый раздел «Конструктор» / «Мои темы»: CRUD тем и заданий |
| `TestsPicker` (ученик) | Третья вкладка **«Темы»** рядом с «По вариантам» / «По заданиям» |
| `HomeworkForm` | Новый пункт `custom_theme`; для `test_by_type` — ручной выбор вариантов |
| `StepProgressDots` | Свободный переход на любой шаг (включая `unseen`) — **для всех тестов** |

## Согласованные решения (2026-06-20)

| Тема | Решение |
|------|---------|
| Проверка ответов | `auto` — авто; `self_check` — кнопка «Сравнить» с полным эталоном |
| Видимость тем | Все ученики преподавателя (`StudentProfile.teacher_id`) |
| Ответ ученика | Текст + **опциональная** загрузка картинки *(обновлено v0.7.9 — см. [`written-homework-photo-submit.md`](written-homework-photo-submit.md))* |
| Лимит «по всем вариантам» в ДЗ | Преподаватель **вручную выбирает варианты** (чекбоксы), не random / не «первые N» |
| Навигация по кружочкам | Разрешена для **всех** тестов (ЕГЭ, кастом, ДЗ) |
| Редактирование | Всегда можно; без версионирования |

## Data model (черновик)

### `teacher_themes`

- `id`, `teacher_id`, `title`, `description` (nullable)
- `is_published` (bool) — видна ученикам
- `sort_order`, `created_at`, `updated_at`

### `custom_tasks`

- `id`, `theme_id`, `title` (nullable), `sort_order`
- `grading_mode`: `auto` | `self_check`
- `question_blocks` (jsonb) — массив блоков: `{ type: "text" | "image", content | url }`
- `reference_answer` (jsonb) — эталон: текст, число, картинки (для `self_check` и подсказки при сравнении)
- `correct_value` (nullable string) — для `auto` (нормализация как в ЕГЭ)
- `created_at`, `updated_at`

### Расширения существующих сущностей

- `TestSession.source`: `exam` | `custom`; `custom_theme_id` (nullable)
- `HomeworkItemKind.CUSTOM_THEME` — `{ kind: "custom_theme", theme_id, task_ids?: uuid[] }`
- `TestByTypeHomeworkItem` — добавить `variants?: string[]` (подмножество вариантов; если пусто — все, как сейчас)

### Uploads

- `POST /api/uploads/images` (teacher + student для ответа)
- Локальное хранение `uploads/` (или `settings.upload_dir`); whitelist MIME, лимит размера
- URL в jsonb-блоках

## Key Assumptions to Validate

- [x] У ученика **один** преподаватель — `StudentProfile.teacher_id` (FK, не nullable).
- [ ] Локальное хранение картинок с лимитом ~5 МБ достаточно для MVP.
- [ ] Редактирование без версий не ломает активные сессии критично (stale до reload).
- [ ] `self_check` не участвует в `score/max_score` сессии (или считается «просмотрено», не «верно/неверно»).

## MVP Scope

### Срез A — Custom Content (ядро)

**IN:**

- CRUD тем и заданий (преподаватель)
- Типы: `auto`, `self_check`
- Контент: текст, число, картинки в вопросе и эталоне; upload
- Вкладка «Темы» в `TestsPicker` для ученика
- `TestSession` с `source=custom`, прохождение в `StepView`
- Ответ ученика: текст + опциональная картинка

**OUT (срез A):**

- Назначение как ДЗ (`custom_theme`)
- Ручной выбор вариантов в `test_by_type`
- Свободные кружочки

### Срез B — Homework

**IN:**

- `HomeworkItemKind.CUSTOM_THEME` в API и `HomeworkForm`
- `test_by_type` + поле `variants[]` — чекбоксы вариантов в UI
- Submit flow для кастомных заданий в ДЗ

### Срез C — UX polish

**IN:**

- `StepProgressDots`: снять `disabled` для `unseen`, переход на любой шаг

## Not Doing (and Why)

| Исключено | Причина |
|-----------|---------|
| Версионирование заданий | Сознательный выбор: «редактировать всегда» |
| Случайная / автоматическая выборка N из пула | Преподаватель выбирает варианты вручную |
| OCR из скриншотов | Преподаватель загружает картинку вручную |
| AI-проверка письменных | `self_check` + сравнение с эталоном |
| Общий банк между преподавателями | Вне scope; темы привязаны к `teacher_id` |
| Варианты ответа A/B/C/D | Можно в v2 |
| S3 / CDN для картинок | Локальный upload в MVP |

## Рассмотренные альтернативы (idea-refine)

| Вариант | Почему не выбран |
|---------|------------------|
| Только числовые задания в MVP | Не покрывает письменные (`self_check`) |
| Отдельный раздел `/student/teacher-tasks` | Пользователь хочет вкладку в «Тестах» |
| `max_steps` со случайной выборкой | Выбран ручной выбор вариантов |
| Версионирование при редактировании | Явно отклонено пользователем |
