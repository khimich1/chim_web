## 1. Executive Summary

Ученики chim_web получают аккаунт от преподавателя и входят по email/паролю на дашборд с четырьмя разделами без объяснения «с чего начать». Это снижает activation и retention между занятиями.

**Цель:** в течение 24 часов после первого входа ≥70% новых учеников совершают одно целевое учебное действие (начать тест, открыть ДЗ или прослушать лекцию).

**Фаза 1 (MVP):** welcome-экран, чеклист первых шагов, подсказка по треку ЕГЭ/ОГЭ, упрощённый первый вход — без кабинета родителя и без публичной регистрации.

## 2. Background & Context

### Текущее состояние

| Компонент | Сейчас |
|-----------|--------|
| Создание аккаунта | Преподаватель через `POST /api/students` |
| Вход | `/login` — email + пароль |
| После входа | Редирект на `/student` — дашборд с quick links |
| Трек | `ege` / `oge` — бейдж на дашборде, без объяснения |
| Прогресс | `ProgressWidget`, `ResumeSessionCards` — для вернувшихся, не для новичков |

### Проблема

- Ученик не знает **первый шаг** (ДЗ? тест? учебник?).
- Технический барьер: email/пароль часто теряются.
- Нет сигнала от преподавателя внутри продукта («начни с диагностики»).
- Custdev (запланирован) должен уточнить приоритеты; PRD опирается на гипотезы из product discovery.

### Почему сейчас

- Платформа минимально жизнеспособна (ДЗ, тесты, учебник, рейтинг).
- Планируется рост базы учеников и SEO-трафик → больше «холодных» первых входов.
- Онбординг — дешёвый рычаг до крупных фич (оплата, родительский кабинет).

### Исследования

- [Custdev-скрипт](../research/custdev-interview-script.md) — 5–7 интервью с учениками.
- [Opportunity Solution Tree](../strategy/product-discovery.md#3-opportunity-solution-tree--онбординг) — outcome: activation 70% / 24ч.

## 3. Objectives & Success Metrics

### Goals

1. **Activation:** новый ученик выполняет ≥1 целевое действие в первые 24 часа.
2. **Clarity:** ученик понимает назначение основных разделов без устного объяснения на занятии.
3. **Retention D7:** ≥50% новых учеников заходят повторно в течение 7 дней.

### Non-Goals (явно вне скоупа фазы 1)

| Non-goal | Почему |
|----------|--------|
| Самостоятельная регистрация (sign-up) | Аккаунты создаёт преподаватель; публичный signup — фаза SEO/freemium |
| Кабинет родителя | Отдельный PRD |
| Полноценный product tour (5+ экранов) | Риск «пролистали и забыли»; начинаем с одного welcome |
| Мобильное приложение | Адаптивный web достаточен для MVP |
| Онбординг преподавателя | Отдельная задача; фокус — ученик |

### Success Metrics

| Метрика | Baseline | Target | Как мерить |
|---------|----------|--------|------------|
| Activation rate (24ч) | TBD после 2 недель логов | ≥70% | Событие `onboarding.first_action` в течение 24ч от `auth.first_login` |
| Completion welcome | — | ≥85% | `onboarding.welcome_completed` / новые ученики |
| Checklist completion (3 шага) | — | ≥60% за 7 дней | `onboarding.checklist_step` × 3 |
| D7 retention | TBD | ≥50% | Повторный `session.start` на день 2–7 |
| Support requests «не могу войти» | Качественно | Снижение | Заметки / Telegram |

**Целевое действие (first_action):** любое из:
- `test.session_started`
- `homework.opened`
- `textbook.topic_opened` (+ прогресс >10% или audio play)

## 4. Target Users & Segments

### Primary: Ученик-новичок

| Атрибут | Описание |
|---------|----------|
| Возраст | 14–18 лет |
| Экзамен | ЕГЭ или ОГЭ (трек в профиле) |
| Контекст | Уже ходит на занятия или только что записался |
| Устройство | Чаще смартфон; иногда ноутбук |
| Техграмотность | Средняя; пароли часто забываются |

### Secondary: Преподаватель (косвенный пользователь)

- Создаёт ученика и сообщает доступы.
- Может назначить «первое действие» (диагностический тест).

### Out of scope для фазы 1

- Самостоятельные ученики с SEO (без занятий).
- Родители.

## 5. User Stories & Requirements

### P0 — Must Have (фаза 1, ~2–3 недели разработки)

| # | User Story | Acceptance Criteria |
|---|-----------|---------------------|
| US-1 | Как **новый ученик**, я хочу **увидеть приветствие при первом входе**, чтобы понять, что делать дальше | Показывается один раз при `first_login`; имя/трек; одна primary CTA; кнопка «Пропустить»; после завершения не показывается снова |
| US-2 | Как **ученик**, я хочу **одну рекомендованную кнопку «Начать»**, чтобы не выбирать из четырёх разделов | CTA ведёт на: (a) активное ДЗ, или (b) диагностический тест, или (c) первую тему учебника — приоритет задаётся правилом (см. §6) |
| US-3 | Как **ученик**, я хочу **чеклист из 3 шагов**, чтобы отслеживать прогресс знакомства | Шаги: «Войти» (auto), «Пройти диагностику / открыть ДЗ», «Изучить одну тему»; прогресс на дашборде до 100%; сворачивается после completion |
| US-4 | Как **ученик**, я хочу **понять свой трек ЕГЭ/ОГЭ**, чтобы знать, какой контент мне показывают | На welcome — 1–2 предложения + бейдж; ссылка «Подробнее» (модалка или tooltip) |
| US-5 | Как **система**, я хочу **фиксировать состояние онбординга**, чтобы не показывать welcome повторно | Поле в БД: `onboarding_completed_at`, `onboarding_checklist` (JSON) или отдельная таблица; API `GET/PATCH /api/students/me/onboarding` |
| US-6 | Как **преподаватель**, я хочу **видеть, кто не завершил онбординг**, чтобы напомнить на занятии | В списке учеников — индикатор «не активирован» / дата последнего входа |

### P1 — Should Have (фаза 1.5, +1–2 недели)

| # | User Story | Acceptance Criteria |
|---|-----------|---------------------|
| US-7 | Как **ученик**, я хочу **войти по ссылке из Telegram**, чтобы не вводить пароль | Magic link / one-time token; TTL 24ч; fallback на пароль |
| US-8 | Как **преподаватель**, я хочу **назначить первое задание при создании ученика**, чтобы CTA был персональным | Поле `first_action` в `StudentCreate`: `diagnostic_test` / `homework` / `textbook_topic` |
| US-9 | Как **ученик**, я хочу **получить короткое видео от преподавателя**, чтобы понять зачем платформа | Встраивание YouTube/VK или `<video>` на welcome; опционально |

### P2 — Nice to Have / Future

| # | User Story | Acceptance Criteria |
|---|-----------|---------------------|
| US-10 | Как **ученик**, я хочу **подсказки при первом посещении раздела** | Coach marks один раз на `/student/tests`, `/homework`, etc. |
| US-11 | Как **ученик**, я хочу **бейдж за первый вход** | Геймификация; интеграция с activity_service |
| US-12 | Как **родитель**, я хочу **получить инструкцию по входу** | Email/Telegram шаблон — вне app |

## 6. Solution Overview

### 6.1 User Flow (фаза 1)

```
[Преподаватель создаёт ученика]
        ↓
[Ученик получает доступ в Telegram/на занятии]
        ↓
[Login /login] → JWT cookie
        ↓
[Проверка: onboarding_completed?]
   Нет → [/student/welcome]
   Да  → [/student]
        ↓
[Welcome Screen]
  - Привет, {имя/email}
  - Трек: ЕГЭ / ОГЭ (кратко)
  - «Здесь ты будешь: решать ДЗ, слушать лекции, проходить тесты»
  - Primary CTA: «Начать диагностику» (или персональный)
  - Secondary: «Позже» → дашборд с чеклистом
        ↓
[Целевое действие] → event first_action
        ↓
[/student] + ChecklistWidget (3 шага, 1–2 закрыты)
```

### 6.2 Правило выбора Primary CTA

| Условие | CTA |
|---------|-----|
| Есть назначенное ДЗ (`assigned`) | «Открыть задание: {title}» |
| Преподаватель указал `first_action` | По указанию |
| Иначе | «Начать диагностический тест» → первый вариант трека |

### 6.3 UI-компоненты (frontend)

| Компонент | Путь | Описание |
|-----------|------|----------|
| `WelcomeScreen` | `app/student/welcome/page.tsx` | Полноэкранный welcome, server + client |
| `OnboardingChecklist` | `components/student/OnboardingChecklist.tsx` | Блок на дашборде, скрывается после 3/3 |
| `TrackExplainer` | `components/student/TrackExplainer.tsx` | Блок объяснения ege/oge |
| `StudentOnboardingGate` | `components/student/StudentOnboardingGate.tsx` | Редирект на welcome (вместо `middleware.ts`) |
| `WelcomeActions` | `components/student/WelcomeActions.tsx` | CTA и «Позже» |

### 6.4 Backend

| Endpoint | Метод | Описание |
|----------|-------|----------|
| `/api/students/me/onboarding` | GET | Статус чеклиста, `completed_at` |
| `/api/students/me/onboarding` | PATCH | Отметить шаг / завершить welcome |
| Расширение `StudentProfile` | migration | `onboarding_state JSONB`, `first_login_at` |

**События (activity / analytics):** — реализовано в `activity_service`, 0 баллов, idempotent по `(student, type, ref_id)`:

| Событие | `ActivityEventType` | ref_id |
|---------|---------------------|--------|
| Просмотр welcome | `onboarding_welcome_viewed` | `welcome` |
| Завершил welcome (CTA) | `onboarding_welcome_completed` | `welcome` |
| Пропустил welcome | `onboarding_welcome_skipped` | `welcome` |
| Шаг чеклиста | `onboarding_checklist_step` | `login` / `first_action` / `lecture` |
| Первое действие | `onboarding_first_action` | `test_session`, `homework`, `checklist`, … |

**Запрос activation rate (24ч):** события `onboarding_first_action` или `step_correct` в течение 24ч после `first_login_at` (SQL — см. `docs/specs/onboarding-metrics.md` при необходимости).

### 6.5 Технические решения

- Состояние онбординга — в **app DB** (PostgreSQL), не в localStorage (мультидевайс).
- Welcome показывается по `first_login_at IS NULL` или `onboarding_completed_at IS NULL`.
- Существующие ученики: миграция помечает `onboarding_completed_at = now()` чтобы не показать welcome старой базе.
- Magic link (P1): таблица `login_tokens` (token_hash, user_id, expires_at, used_at).

### 6.6 Wireframe (текстовый)

```
┌─────────────────────────────────────┐
│  chim_web · Кабинет ученика         │
├─────────────────────────────────────┤
│  Привет!                            │
│  Ты готовишься к [ЕГЭ по химии]     │
│                                     │
│  Здесь между занятиями ты:          │
│  · делаешь домашние задания         │
│  · слушаешь лекции                  │
│  · тренируешься на тестах           │
│                                     │
│  ┌─────────────────────────────┐   │
│  │   Начать диагностику        │   │
│  └─────────────────────────────┘   │
│         Позже, посмотрю сам         │
└─────────────────────────────────────┘

Дашборд (после):
┌─ Первые шаги ─────────── 1/3 ─────┐
│ ✓ Войти в кабинет                   │
```

## 7. Assumptions & Risks

| Assumption | Риск | Валидация |
|------------|------|-----------|
| Главный барьер — незнание первого шага | Средний | Custdev US-3, US-5 |
| 70% activation достижимо за 24ч | Средний | A/B после релиза |
| Диагностический тест — лучший первый CTA | Высокий | Custdev + сравнение с ДЗ-first |
| Ученики не пролистают welcome | Средний | Метрика skip rate <30% |
| Magic link нужен в фазе 1 | Низкий | Отложить в P1 если пароль не болит |

**Pre-mortem (кратко):**
- Welcome раздражает опытных → «Пропустить» + не показывать старой базе.
- Чеклист визуальный шум → сворачивать после completion.
- Преподаватель не сообщает про платформу → инструкция в Telegram-шаблоне.
